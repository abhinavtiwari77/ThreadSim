Threading Models Explained

Many-to-One (M:1)

Maps many user-level threads to a single kernel thread. Thread management is done in user space, which is efficient but can block the entire process if one thread performs a blocking system call.

One-to-Many (1:N)

Maps each user thread to many kernel threads. This model allows multiple threads to run in parallel on multiprocessors and continue running if a thread makes a blocking system call.

Many-to-Many (M:N)

Multiplexes many user-level threads to a smaller or equal number of kernel threads. This provides the best of both worlds: efficiency of M:1 and the parallelism of 1:N.

Thread States

  • Ready: Thread is waiting to be assigned to a CPU
  • Running: Thread is currently being executed
  • Blocked: Thread is waiting for an I/O operation to complete
  • Terminated: Thread has completed execution
